android 反编译与重新编译签名（一）

环境准备

jdk , android sdk , ndk , android studio , android 真机或模拟器 。

编写一个 android 程序，并生成一个 apk 文件。

以下操作均在 windows 环境中。

反编译 apk 文件

首先下载 apktool 工具, apktool 主要作用是反编译 apk 和重新打包签名 apk ，具体功能用法可看其官网介绍，下载好 apktoolx.x.x.jar 将其重命名为 apktool.jar ，再在同目录下创建一个文件 apktool.bat 文件，在文件中写入如下 Windows 的批处理： 

@echo off
if "%PATH_BASE%" == "" set PATH_BASE=%PATH%
set PATH=%CD%;%PATH_BASE%;
java -jar -Duser.language=en "%~dp0\apktool.jar" %*

以上作用的目的在于后续使用的时候不用带上 jar 后缀。
再将该目录配置到环境变量中即可。

接下来打开 cmd 切换到我们的 apk 文件所在目录输入一条命令即可反编译 apk：

C:\Users\xxxxxx\Desktop\xxxxxx>apktool d app-debug.apk
I: Using Apktool 2.2.4 on app-debug.apk
I: Loading resource table...
I: Decoding AndroidManifest.xml with resources...
I: Loading resource table from file: C:\Users\jiuyou\AppData\Local\apktool\frame
work\1.apk
I: Regular manifest package...
I: Decoding file-resources...
I: Decoding values */* XMLs...
I: Baksmaling classes.dex...
I: Copying assets and libs...
I: Copying unknown files...
I: Copying original files...

如你所见反编译 apk 的基本语法就是 apktool d <xxx.apk>

如果提示：

‘apktool’ 不是内部或外部命令，也不是可运行的程序
或批处理文件。

说明你的环境变量或者批处理文件没有配置正确。

如果执行成功，目录下回出现一个文件夹，里面的内容大概是下面这样：

original
res         //程序的所有资源文件
smali       //程序的反汇编代码
AndroidManifest.xml     //不用解释了吧...
apktool.yml

重新编译并签名 apk

使用 apktool b <dir> 命令重新编译 apk ，dir 指 之前反编译出来的文件夹：

C:\Users\xxxxxx\Desktop\xxxxxx>apktool b app-debug
I: Using Apktool 2.2.4
I: Checking whether sources has changed...
I: Checking whether resources has changed...
I: Building apk file...
I: Copying unknown files/dir...

编译成功后会在 app-debug 目录下 的 dist 下生成编译好的 apk，但此时的 apk 还没有签名，无法使用，接下来需要用到 signapk.jar 工具对其进行签名， signapk.jar 是 android 源码中的一个签名工具，代码位于 android源码目录下的 /build/tools/signapk/SignApk.java 文件中，使用 signapk.jar 签名还需要 android 源码中提供的签名文件 testkey.pk8 和 testkey.x509.pem，在 /build/tagget/product/security 目录中，将这三个文件都放到之前 apktool.jar 文件所在目录中（这个目录应该是添加到了环境变量中的），再在目录下创建一个文件 signapk.bat ，写入以下内容：

java -jar "%~dp0signapk.jar" "%~dp0testkey.x509.pem" "%~dp0testkey.pk8" %1 signed.apk

作用是为了方便使用 signapk.jar 来进行签名。

现在再回到之前重新编译好的 apk 目录下对其进行签名：

C:\Users\xxxxxx\Desktop\xxxxxx\app-debug\dist>signapk app-debug.apk

C:\Users\xxxxxx\Desktop\xxxxxx\app-debug\dist>java -jar "D:\tools\signapk.j
ar" "D:\tools\testkey.x509.pem" "D:\tools\testkey.pk8" app-debug.apk signed.apk

完成后目录下就会出现已经签名好的 apk 文件 signed.apk。

最后安装测试 apk 是否可以运行。

反编译和重新编译签名就说完，到目前已经可以替换一些资源文件并重新打包 apk 了。

关于用到的工具

apktool.jar 还是很方便下载和使用的，但是 signapk.jar 也不用真的去下载 android 源码来编译的，网上都可以找到，也可以直接搜索 [Android逆向助手_v2.x] 工具包，以上说到的工具都有包含，并做了封装。
