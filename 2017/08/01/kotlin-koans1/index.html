fun start(str:String = "OK") = str

object start{

第二经 Java to kotlin conversion

We have a handy tool for Java developers: Java to Kotlin converter. To become familiar with it, please convert the java function toJSON in the file JavaCode.java. You can just copy-paste the code to Kotlin file and agree to automatically convert it to Kotlin.

我们为 Java 开发人员提供了一个工具（Java to Kotlin converter），你现在只需要将 JavaCode.java 中的 toJSON 方法复制到 kotlin 文件中，并同意将其转换为 java 代码即可。

转换后的结果：

fun toJSON(collection: Collection<Int>): String {
    val sb = StringBuilder()
    sb.append("[")
    val iterator = collection.iterator()
    while (iterator.hasNext()) {
        val element = iterator.next()
        sb.append(element)
        if (iterator.hasNext()) {
            sb.append(", ")
        }
    }
    sb.append("]")
    return sb.toString()
}

check 以下，的确可以通过 。

ps： 『Java to Kotlin converter』 虽然好用，但不能过于依赖。

本题关键字 『Java to Kotlin converter』

第三经 Named arguments

Default and named arguments help to minimize the number of overloads and improve the readability of the function invocation. The library function joinToString is declared with default values for parameters:

fun joinToString(

separator: String = “, “,

prefix: String = “”,

postfix: String = “”,

//…

): String

It can be called on a collection of Strings. Specifying only two arguments make the function joinOptions() >return the list in a JSON format (e.g., “[a, b, c]”)

默认参数和具名参数可以减少方法重载的次数，提高函数调用的可读性。kotlin 库中自带的 joinToString 函数参数就设置了默认值。扩展函数 joinToString 可以在字符串集合上直接调用，指定两个参数，使 joinOptions 函数返回 JSON 格式 (e.g., “[a, b, c]”)

fun joinOptions(options: Collection<String>) = options.joinToString(prefix = "[",postfix = "]")

prefix , postfix 参数并不是第一个和第二个参数，但我们使用了具名参数的方法，就可以无视参数顺序，同时提高了可读性

本题关键字 『具名参数』

第四经 Default arguments

现有 Java 代码如下：

public String foo(String name, int number, boolean toUpperCase) {

return (toUpperCase ? name.toUpperCase() : name) + number;

}

public String foo(String name, int number) {

return foo(name, number, false);

}

public String foo(String name, boolean toUpperCase) {

return foo(name, 42, toUpperCase);

}

public String foo(String name) {

return foo(name, 42);

}

使用 kotlin 默认参数的形式修改上述代码：

fun foo(name: String, number: Int=42, toUpperCase: Boolean=false) =
        (if (toUpperCase) name.toUpperCase() else name) + number

由此可见， Java 中方法重载的写法在 kotlin 中可以被默认参数取代了（至少大多数情况下可以了）

本地关键字 『默认参数』

第五经 Lambdas

编写一个方法 containsEven(collection: Collection<Int>): Boolean ，当集合中存在一个偶数时返回 true ，否则 返回 false。

java style:

fun containsEven(collection: Collection<Int>): Boolean {
    for(item in collection){
        if(item%2==0){
            return true
        }
    }
    return false
}

kotlin style:

fun containsEven(collection: Collection<Int>): Boolean = collection.any { it % 2 == 0 }

any 函数内部帮我们封装了循环，它接收一个 lambda 表达式 ，其参数就是 item，其返回类型是 Boolean。

public inline fun <T> Iterable<T>.any(predicate: (T) -> Boolean): Boolean {
    for (element in this) if (predicate(element)) return true
    return false
}

本题关键字 『lambda』

第六经 Strings

学习 kotlin 中的原始字符串和字符串模板

原始字符串

原始字符串由三重引号（“”“）分隔，不包含转义，并且可以包含换行符和任何其他字符，比如可以像下面这样：

val text = """
    for (c in "foo")
        print(c)
"""

注意，每行前面的空格也是字符串的一部份。
可以使用 trimMargin() 函数删除前导空格：

val text = """
    |Tell me and I forget.
    |Teach me and I remember.
    |Involve me and I learn.
    |(Benjamin Franklin)
    """.trimMargin()

上面的 text 如果打印出来是下面这样：

Tell me and I forget.
Teach me and I remember.
Involve me and I learn.
(Benjamin Franklin)

默认 | 用作边距前缀，但您可以选择另一个字符，并将其作为参数传递，如 trimMargin(">")。

字符串模板

字符串可以包含模板表达式，模板表达式的结果会自动连接到字符串中，模板表达式以 $ 符号开头，后面跟一个简单的变量名：

val i = 10
val s = "i = $i" // s = "i = 10"

如果 $ 符号后跟上 {} ，则可以写任何表达式：

val s = "abc"
val str = "$s.length is ${s.length}" // str = "abc.length is 3"

原始字符串和普通字符串内部都支持模板。如果您需要在原始字符串（不支持反斜杠转义）中表示一个文字 $ 字符，则可以使用以下语法：

val price = """
${'$'}9.99
"""

// $9.99

回归正题
如何用正则表达式匹配 13 JUN 1992 这样的日期：

java style:

val month = "(JAN|FEB|MAR|APR|MAY|JUN|JUL|AUG|SEP|OCT|NOV|DEC)"

fun getPattern(): String {
    return "\\d{2} " + month +" \\d{4}"
}

如果是 java 需要用到 反斜杠转义 和 + 号，在正则表达式复杂的时候可读性很差，当然变量可以使用 String.format() 替换掉，但是 反斜杠转义呢？

看看 kotlin style:

val month = "(JAN|FEB|MAR|APR|MAY|JUN|JUL|AUG|SEP|OCT|NOV|DEC)"

fun getPattern(): String ="""\d{2} ${month} \d{4}"""

第七经 Data classes

在 java 中 一个实体类大概长这样：

public class Person {

private final String name;

private final int age;

public Person(String name, int age) {

    this.name = name;

    this.age = age;

}

public String getName() {

    return name;

}

public int getAge() {

    return age;

}

}

在 kotlin 中实体类只需要用 data 关键字 标识 class：

data class Person(val name:String,val age:Int)

注意

data class 生成的实体类没有默认的无参构造方法，如果需要生成默认无参构造方法需要使用插件 No-arg
data class 生成的实体类是 final 类型的，无法被继承，要想被继承需要 All-open

第八经 Nullable types

阅读 关于Kotlin中的空安全和安全调用，并使用只有 if 表达式重写以下Java代码：

public void sendMessageToClient(

@Nullable Client client,

@Nullable String message,

@NotNull Mailer mailer

) {

if (client == null || message == null) return;

PersonalInfo personalInfo = client.getPersonalInfo();

if (personalInfo == null) return;

String email = personalInfo.getEmail();

if (email == null) return;

mailer.sendMessage(email, message);

fun sendMessageToClient(

第九经 Smart casts

阅读 智能转换 和 when 表达式，重写以下 Java 代码

public int eval(Expr expr) {
if (expr instanceof Num) {

    return ((Num) expr).getValue();

}
if (expr instanceof Sum) {
    Sum sum = (Sum) expr;
    return eval(sum.getLeft()) + eval(sum.getRight());
}
throw new IllegalArgumentException("Unknown expression");
}

kotlin style :

fun eval(expr: Expr): Int =
        when (expr) {
            is Num -> expr.value
            is Sum -> eval(expr.left)+eval(expr.right)
            else -> throw IllegalArgumentException("Unknown expression")
        }

interface Expr
class Num(val value: Int) : Expr
class Sum(val left: Expr, val right: Expr) : Expr

第十经 Extension functions

阅读 扩展函数。编写扩展函数 Int.r() 和 Pair.r() 使它们将 Int 和 Pair 转换成 RationalNumber

fun Int.r(): RationalNumber = RationalNumber(this, 1)
fun Pair<Int, Int>.r(): RationalNumber = RationalNumber(first, second)

data class RationalNumber(val numerator: Int, val denominator: Int)

第十一经 Object expressions

kotlin 中的 对象表达式 和 java 中的匿名类扮演着相同的角色。
本题为 用对象表达式的方式给 Collections.sort 设置比较器参数，降序排列传入的集合：

import java.util.*

fun getList(): List<Int> {
    val arrayList = arrayListOf(1, 5, 2)
    Collections.sort(arrayList, object : Comparator<Int> {
    override fun compare(x: Int, y: Int) = y - x
})
    return arrayList
}

第十二经 SAM conversions

当对象的接口只具有一个单抽象方法时，可以用 lambda 表达式来替换它，详细请看 SAM-转换
这里我们将上一经的 对象表达式修改为 lambda 表达式：

import java.util.*

fun getList(): List<Int> {
    val arrayList = arrayListOf(1, 5, 2)
    Collections.sort(arrayList, { x, y -> y-x })
    return arrayList
}

第十三经 Extensions on collections

Kotlin 代码可以很容易地与 Java 代码混编。 因此，在 Kotlin 中，我们不介绍我们自己的集合，而是使用标准的Java（稍做改进）。在 Kotlin 标准库中有很多扩展功能，使得集合的工作更加方便。 再次使用 sortedDescending 的扩展函数重写上一个示例。

fun getList(): List<Int> {
    return arrayListOf(1, 5, 2).sortedDescending()
}
