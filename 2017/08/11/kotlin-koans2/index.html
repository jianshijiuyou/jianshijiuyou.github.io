第十四经 Comparison

阅读 操作符重载，学习如何在 kotlin 中 自定义 ==, <, + 等操作符的用法，然后让我们自定义的 Date 类可以比较大小。

让自定义的类拥有比较(<,>,<=,>=)的功能需要实现的操作符方法为 compareTo，所以这里我们要给 MyDate 类中加上 compareTo 方法：

fun main(args: Array<String>) {
    val date1=MyDate(2017,1,1)
    val date2=MyDate(2017,8,2)
    println(date1>date2)
}


data class MyDate(val year: Int, val month: Int, val dayOfMonth: Int) {
    operator fun compareTo(other: MyDate) = when {
        year != other.year -> year - other.year
        month != other.month -> month - other.month
        else -> dayOfMonth - other.dayOfMonth
    }
}

第十五经 In range

在 kotlin 中 判断或检查对象 表现为 contains 方法的调用。
举个栗子：

val list = listOf("a","b")
"a" in list   //实际上等价于  list.contains("a")
"a" !in list  //实际上等价于  !list.contains("a")

阅读相关文章 ranges ，然后在我们的类 DateRange 中添加 contains 方法，让它也具有判断检查日期的功能：
这里需要用到上一经讲的 MyDate 类

class DateRange(val start: MyDate, val endInclusive: MyDate) {
    operator fun contains(item: MyDate): Boolean = start <= item && item <= endInclusive
}

fun checkInRange(date: MyDate, first: MyDate, last: MyDate): Boolean {
    return date in DateRange(first, last)
}

第十六经 Range to

我们现在已经可以用 date in DateRange(first, last) 这样的方法来判断自定义的 MyDate 类了，但是还不够简洁，在 kotlin 中
可以 value in 1..10 这样判断一个数字是否在给定区间，我希望我们的 MyDate 类也能够实现 date in startDate..endDate 这样的效果，可以吗？当然可以！
.. 对应的操作符方法是 rangeTo ， rangeTo 方法需要回返一个 ClosedRange ，什么是 ClosedRange 呢?

ClosedRange 是 kotlin 提供的一个接口，用来帮助我们确定一个类型的区间范围的，在这里就是 rangeTo 用 ClosedRange 来确定 start..end 表示的范围，下面看实现：

operator fun MyDate.rangeTo(other: MyDate) = DateRange(this, other)

class DateRange(override val start: MyDate, override val endInclusive: MyDate): ClosedRange<MyDate>

fun checkInRange(date: MyDate, first: MyDate, last: MyDate): Boolean {
    return date in first..last
}

第十七经 For loop

先看个栗子，打印 1 到 10 的数字：

for(i in 1..10){
    println(i)
}

现在我想自己实现下面的效果：
打印字母 a,b,c,d 一直到 z

for(i in "a".."z"){
    println(i)
}

那么如何实现这种效果呢，请阅读 for-循环。
首先还是要实现 ragneTo 操作符，这次 ragneTo 需要返回的是 Iterable 接口，这样程序才知道每次迭代的数据是什么,Iterable 接口就一个方法 需要返回一个 Iterator ，Iterator 中有两个眼熟的方法，hasNext 和 next，看到这大家都应该明白原理了吧～

fun main(args: Array<String>) {
    for (i in "a".."z") {
        println(i)
    }

}

operator fun String.rangeTo(s: String) = LetterRange(this,s)

class LetterRange(val start:String,val end:String) : Iterable<String> {
    override fun iterator() = LetterIterator(start,end)
}

class LetterIterator(var start:String,var end:String) : Iterator<String> {
    private var cur=start

    override fun hasNext(): Boolean {

        return start[0]<=end[0]
    }

    override fun next(): String {
        cur=start
        start=(start[0]+1).toString()
        return cur
    }

}

第十八经 Operators overloading

重载合适的操作符，实现如下效果：

 "a" * 3    // 结果是  "aaa"
 "abcd" - "c"  // 结果是  "abd"

相关知识点可以阅读 操作符重载 

fun main(args: Array<String>) {
    println("a" * 5)
    println("abcd" - "c")
}

operator fun String.minus(str:String)=this.replace(str,"")

operator fun String.times(num:Int):String{
    var res=""
    for (i in 1..num){
        res+=this
    }
    return res
}

第十九经 Destructuring declarations（解构声明）

有时把一个对象 解构 成很多变量会很方便，例如:

val (name, age) = person

这种语法称为 解构声明 。一个解构声明同时创建多个变量。 我们已经声明了两个新变量：name 和 age，并且可以独立使用它们： 

println(name)
println(age)

一个解构声明会被编译成以下代码：

val name = person.component1()
val age = person.component2()

####在 kotlin 中 数据类自动声明 componentN() 函数
也就是说数据类默认就可以被解构

更多关于解构的内容请阅读 解构声明

第二十经 Invoke

重载合适操作符，实现如下效果：

"a"(5)    // 结果为 "aaaaa"

圆括号转换为调用带有适当数量参数的 invoke，所以这里需要实现 invoke 操作符：

fun main(args: Array<String>) {
    println("a"(5))
}

operator fun String.invoke(num:Int):String{
    var res=""
    for (i in 1..num){
        res+=this
    }
    return res
}
